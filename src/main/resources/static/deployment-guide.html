Prerequisites

🖥 Local Machine

  • JDK 17 or higher — Java 17, 21, or 23 all work
    java -version
  • Maven 3.6+ — or use the included mvnw wrapper
  • Git — for cloning and version control
  • IDE — IntelliJ IDEA, VS Code with Java extensions, or Eclipse

☁ Cloud / Production

  • GitHub account — for source repository
  • Railway account — railway.app (free tier available)
  • Supabase account — supabase.com (free PostgreSQL)
  • Alternatively: any cloud with Java runtime (Render, Fly.io, Heroku, AWS, GCP)
ℹ
The project includes mvnw and mvnw.cmd — Maven wrappers. You do not need Maven installed globally. Just use ./mvnw (Linux/Mac) or mvnw.cmd (Windows).

Local Development Setup

1

Clone the repository

git clone https://github.com/suraj-suryn/task-scheduler.git
cd task-scheduler
2

Set JAVA_HOME (if JDK not on PATH)

Windows (PowerShell):

$env:JAVA_HOME = "C:\Program Files\Java\jdk-23"
$env:PATH = "$env:JAVA_HOME\bin;$env:PATH"
java -version   # confirm version

Linux / macOS:

export JAVA_HOME=/usr/lib/jvm/java-17-openjdk
export PATH=$JAVA_HOME/bin:$PATH
3

Run the application (local H2 database — no setup needed)

Windows:

.\mvnw.cmd spring-boot:run

Linux / macOS:

./mvnw spring-boot:run

App starts at http://localhost:8080

H2 console at http://localhost:8080/h2-console

4

First login — create your admin account

  • Go to http://localhost:8080/register
  • Register a user — by default all users get ROLE_USER
  • To promote to admin: open H2 Console at /h2-console, connect with JDBC URL jdbc:h2:file:./data/tasksdb, run: UPDATE users SET role = 'ROLE_ADMIN' WHERE username = 'yourname';

Database Setup — H2 (Local) & PostgreSQL (Production)

💾 H2 — Local Development

H2 is an embedded in-memory/file database included in the project. No installation required.

The local database file is saved at ./data/tasksdb.mv.db relative to where you run the app. Data persists across restarts.

H2 Console URL: http://localhost:8080/h2-console

FieldValue
JDBC URLjdbc:h2:file:./data/tasksdb
Usernamesa
Password(blank)

🐘 PostgreSQL — Production

PostgreSQL is used for cloud deployments. The project already includes the PostgreSQL JDBC driver in pom.xml.

Activate via Spring profile prod:

SPRING_PROFILES_ACTIVE=prod

When the prod profile is active, application-prod.properties overrides the datasource to use PostgreSQL with connection string from DATABASE_URL env var.

Supabase PostgreSQL Configuration

Supabase provides a free-tier managed PostgreSQL database. Follow these steps to create and connect your database.

1

Create a Supabase project

  • Go to https://supabase.com → Sign up / Log in
  • Click New Project
  • Choose your organization, give the project a name (e.g. task-scheduler)
  • Set a strong Database Password — save it, you'll need it
  • Select a region closest to your deployment (e.g. ap-southeast-1 for Asia)
  • Click Create new project — takes ~2 minutes
2

Get the connection string

  • In your Supabase project → left sidebar → Project Settings → Database
  • Scroll to Connection string section
  • Select tab: URI
  • Copy the JDBC-compatible URL — it looks like:
    jdbc:postgresql://aws-0-ap-southeast-1.pooler.supabase.com:5432/postgres?sslmode=require
  • Also note: User = postgres.<project-ref>, Password = what you set above
⚠
Use the Session pooler connection (port 5432) for Spring Boot, not the Transaction pooler (port 6543). The Transaction pooler does not support JPA prepared statements well.
3

Schema creation — automatic

Spring Boot with spring.jpa.hibernate.ddl-auto=update will automatically create all tables on first startup. You do not need to run any SQL scripts manually.

Tables created: users, tasks, categories

4

Create the first admin user via SQL (after first startup)

Go to Supabase → SQL Editor → run:

-- Promote a user to ROLE_ADMIN after they register
UPDATE users SET role = 'ROLE_ADMIN' WHERE username = 'your-username';

Application Properties Configuration

📄 application.properties — Default (Local / H2)

# ─── SERVER ───────────────────────────────────────────
server.port=8080

# ─── THYMELEAF ────────────────────────────────────────
spring.thymeleaf.cache=false   # set true in production for performance

# ─── H2 DATABASE (local dev) ──────────────────────────
spring.datasource.url=jdbc:h2:file:./data/tasksdb;DB_CLOSE_DELAY=-1;DB_CLOSE_ON_EXIT=FALSE
spring.datasource.username=sa
spring.datasource.password=
spring.datasource.driver-class-name=org.h2.Driver

# ─── JPA / HIBERNATE ──────────────────────────────────
spring.jpa.database-platform=org.hibernate.dialect.H2Dialect
spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=false

# ─── H2 CONSOLE ───────────────────────────────────────
spring.h2.console.enabled=true
spring.h2.console.path=/h2-console

# ─── APP BASE URL (for email verification links) ──────
app.base-url=http://localhost:8080

# ─── SPRINGDOC / SWAGGER ──────────────────────────────
springdoc.packagesToScan=com.suraj.scheduler.controller
springdoc.pathsToMatch=/api/**

📄 application-prod.properties — Production (PostgreSQL)

Create this file at src/main/resources/application-prod.properties. It is activated by setting SPRING_PROFILES_ACTIVE=prod.

# ─── SERVER ───────────────────────────────────────────
server.port=${PORT:8080}   # Railway injects PORT env var automatically

# ─── THYMELEAF ────────────────────────────────────────
spring.thymeleaf.cache=true

# ─── POSTGRESQL DATABASE ──────────────────────────────
spring.datasource.url=${DATABASE_URL}
spring.datasource.username=${DB_USERNAME}
spring.datasource.password=${DB_PASSWORD}
spring.datasource.driver-class-name=org.postgresql.Driver

# ─── CONNECTION POOL (HikariCP) ───────────────────────
spring.datasource.hikari.maximum-pool-size=5
spring.datasource.hikari.minimum-idle=1
spring.datasource.hikari.connection-timeout=20000
spring.datasource.hikari.idle-timeout=300000

# ─── JPA / HIBERNATE ──────────────────────────────────
spring.jpa.database-platform=org.hibernate.dialect.PostgreSQLDialect
spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=false

# ─── DISABLE H2 CONSOLE in production ─────────────────
spring.h2.console.enabled=false

# ─── APP BASE URL ─────────────────────────────────────
app.base-url=${APP_BASE_URL:https://your-app.up.railway.app}

# ─── EMAIL ────────────────────────────────────────────
spring.mail.host=${MAIL_HOST:smtp.gmail.com}
spring.mail.port=${MAIL_PORT:587}
spring.mail.username=${MAIL_USERNAME:}
spring.mail.password=${MAIL_PASSWORD:}
spring.mail.properties.mail.smtp.auth=true
spring.mail.properties.mail.smtp.starttls.enable=true
🔒
Never commit credentials to git. Use ${ENV_VAR_NAME} placeholders in properties files. Real values go in Railway / your cloud platform's environment variables.

Railway Cloud Deployment — Step-by-Step

1

Push your code to GitHub

git init                          # if not already a git repo
git add -A
git commit -m "initial commit"
git remote add origin https://github.com/YOUR_USERNAME/task-scheduler.git
git push -u origin master
2

Create a Railway account and new project

  • Go to https://railway.app → Sign up with GitHub
  • Click New Project → Deploy from GitHub repo
  • Authorize Railway to access your GitHub account
  • Select your task-scheduler repository
  • Click Deploy Now
3

Configure environment variables in Railway

In your Railway project → click your service → Variables tab → Add each:

NameValue
SPRING_PROFILES_ACTIVEprod

4

Verify the deployment

  • Railway → your service → Deployments tab → watch build logs
  • Look for: Started TaskSchedulerApplication in X.XXX seconds
  • Click the public URL generated by Railway (e.g. https://task-scheduler-production-xxxx.up.railway.app)
5

Set up auto-deploy on push

Railway auto-deploys every time you push to the connected branch (master by default). To change the branch: Settings → Source → Branch.

6

Add a custom domain (optional)

  • Railway → your service → Settings → Domains → Add Custom Domain
  • Enter your domain (e.g. tasks.yourdomain.com)
  • Add the CNAME record shown to your DNS provider
  • Railway provisions SSL automatically

Deploying to Other Platforms

🚀 Render.com

  1. New → Web Service → connect GitHub repo
  2. Runtime: Java (or Docker)
  3. Build Command: ./mvnw clean package -DskipTests
  4. Start Command: java -jar target/*.jar
  5. Add the same environment variables as Railway
  6. Free tier: service sleeps after 15 min of inactivity

🪰 Fly.io

  1. Install flyctl: curl -L https://fly.io/install.sh | sh
  2. fly auth login
  3. In project root: fly launch → select region
  4. Set secrets: fly secrets set DATABASE_URL=... DB_USERNAME=... DB_PASSWORD=... SPRING_PROFILES_ACTIVE=prod
  5. fly deploy

☁ AWS Elastic Beanstalk

  1. Build JAR: ./mvnw package -DskipTests
  2. AWS Console → Elastic Beanstalk → Create application
  3. Platform: Java (Corretto 17)
  4. Upload target/*.jar as the application archive
  5. Configuration → Software → Environment properties → add all env vars

🐳 Docker

Create a Dockerfile in project root:

FROM eclipse-temurin:17-jdk-alpine
WORKDIR /app
COPY target/*.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar"]

Build & run:

docker build -t task-scheduler .
docker run -p 8080:8080 \
  -e SPRING_PROFILES_ACTIVE=prod \
  -e DATABASE_URL=... \
  -e DB_USERNAME=... \
  -e DB_PASSWORD=... \
  task-scheduler

Spring Security & Admin Account Setup

How authentication works

  • Registration at /register creates a ROLE_USER account
  • If email is configured, the user receives a verification email — they must click the link before logging in
  • If email is not configured, accounts are automatically enabled on registration
  • Login at /login with username & password
  • Admin routes (/admin/**) require ROLE_ADMIN

Promote a user to Admin

Option A — Via Supabase SQL Editor:

UPDATE users SET role = 'ROLE_ADMIN' WHERE username = 'your-username';

Option B — Via H2 Console (local):
Connect to jdbc:h2:file:./data/tasksdb and run the same query.

Option C — Once admin exists:
Admin Portal → Manage Users → change role via dropdown.

Troubleshooting Common Issues

!

App starts but shows "Whitelabel Error Page" on /

No mapping for /. Navigate to /dashboard or /login instead. Or check if the app is starting on a different port.

!

PostgreSQL connection refused / SSL error

  • Ensure ?sslmode=require is appended to DATABASE_URL
  • Supabase: use port 5432 (Session pooler), not 6543
  • Check DB_USERNAME format: postgres.your-project-ref (note the dot)
  • Verify DB_PASSWORD has no leading/trailing spaces
!

java.lang.UnsupportedClassVersionError

The JDK version used to build is newer than the JRE used to run. Ensure JAVA_HOME points to JDK 17+. The project requires Java 17 minimum (<java.version>17</java.version> in pom.xml).

!

Railway build fails: "mvnw: Permission denied"

git update-index --chmod=+x mvnw
git commit -m "fix: make mvnw executable"
git push
!

Can't log in after registering (email not enabled)

If email/SMTP is not configured, the verification link is never sent. Fix: connect to the database and run:

UPDATE users SET enabled = true WHERE username = 'your-username';
!

H2 Console shows "Database not found"

Use the full JDBC URL exactly: jdbc:h2:file:./data/tasksdb. The path is relative to where you run the JAR, not the project root. Make sure to run from the project root directory.

!

Supabase "too many connections"

Supabase free tier has a connection limit (~20 total). Reduce HikariCP pool size in application-prod.properties:

spring.datasource.hikari.maximum-pool-size=3
spring.datasource.hikari.minimum-idle=1
