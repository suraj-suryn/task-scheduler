Section 2
Custom DSA Structures

Three hand-written data structures power the core scheduling logic — no third-party scheduling libraries used.

📐 Interval Tree (BST)

IntervalTree

A Binary Search Tree that stores task time intervals and detects scheduling conflicts in O(log n). Before every task save, isOverlapping() is called against all existing tasks. If any time range intersects, a TaskOverlapException is thrown.

Insert: O(log n) Query: O(log n) Used on: every task create/update
🔺 Priority Heap (Min/Max)

PriorityHeap

A custom heap implementation sorted by task priority (1–10). The "What's Next?" smart suggest feature polls the heap's peek to find the highest-priority PENDING task whose all dependencies are completed.

Add: O(log n) Poll: O(log n) Peek: O(1)
🕸 Dependency Graph (DAG + DFS)

DependencyGraph

A Directed Acyclic Graph with DFS-based cycle detection. When adding a task dependency, hasCycle() traverses the graph to ensure no circular dependencies exist. Cycles throw DependencyCycleException.

Cycle check: O(V + E) Topological sort: O(V + E)
All three structures are hydrated from the database at startup via @PostConstruct in TaskServiceImpl, so they stay in sync with persisted task data.

Section 4
Authentication Flow

Spring Security 6 with BCrypt password hashing, form-based login, and role-based access control.

1

Register /register

User enters username, email, and password. Password is BCrypt-hashed before storage. Default role: ROLE_USER.

2

Login /login

Spring Security's DaoAuthenticationProvider validates credentials. On success, redirects to /dashboard.

3

Session Management

Server-side session with CSRF protection. Sessions auto-expire. Logout via POST /logout to prevent CSRF attacks.

4

Role-Based Access

ROLE_USER — own tasks only. ROLE_ADMIN — all tasks + /admin/**. Enforced via Spring Security rules AND @PreAuthorize.

Section 5
Task Management Demo

Navigate to /tasks — full CRUD with overlap prevention, dependency management, and real-time status updates.

1

Creating a Task

Click "Add Task" → modal opens with full form: name, description, priority (1–10), category, start/end time, recurrence, and dependency IDs. On submit, the backend validates:

  • Time validity — end must be after start (InvalidTaskTimeException)
  • Overlap check — IntervalTree scans for conflicts (TaskOverlapException)
  • Cycle check — DependencyGraph DFS validates dependencies (DependencyCycleException)
2

Searching & Filtering

Filter bar supports 3 simultaneous filters: text search (LIKE on task name), status dropdown, and category dropdown. All filters combine with AND logic via JPQL @Query in TaskRepository.

Example: search "login" + status PENDING → shows only matching pending tasks.

3

Updating Status (inline)

Each task row has an inline status <select>. Changing it fires PATCH /api/tasks/{id}/status immediately — no page reload. The scheduler engine also auto-transitions tasks based on time.

4

Recurrence

Tasks can repeat DAILY / WEEKLY / MONTHLY. When the scheduler auto-completes a task, it clones it with the next scheduled time if recurrence is set and the recurrence end date hasn't passed.

Section 6
Dashboard & Analytics

The dashboard at /dashboard gives a real-time overview of all your tasks with visual charts.

📊

Status Donut Chart

Chart.js doughnut — shows breakdown of Pending, Running, Completed, Failed, Overdue tasks. Data fetched from GET /api/tasks on page load.

📈

Priority Band Bar Chart

Chart.js bar — groups tasks into Low (1–3), Medium (4–6), High (7–10) priority bands. Helps identify workload distribution.

🔮

"What's Next?" Smart Suggest

Calls GET /api/tasks/suggest — uses the PriorityHeap DSA to find the highest-priority PENDING task whose ALL dependencies are completed.

📅

Upcoming & Completed Widgets

Shows next 5 upcoming tasks sorted by deadline, and 5 recently completed tasks. Overdue tasks show a warning banner with count.

Section 11
Admin Portal

Accessible only to users with ROLE_ADMIN. Regular users get 403 Forbidden at both UI and API level.

System Stats /admin

Shows totals across all users: total tasks, pending, running, completed, failed, overdue. Plus total registered users.

User Management /admin/users

Table of all users with role dropdown (USER/ADMIN) and Disable/Enable button. Role changes and account status take effect immediately.

Admin can see every task from every user. Can delete any task with confirmation. Useful for moderation and cleanup.

Access Protection

Double-protected: Spring Security rule .requestMatchers("/admin/**").hasAuthority("ROLE_ADMIN") at HTTP level AND @PreAuthorize on controller methods.

Section 14
Deployment Architecture

Railway (Hosting)

  • Auto-detect Dockerfile or Maven build
  • Trigger: git push to master branch
  • Environment variables in dashboard
  • Custom domain or .up.railway.app URL
  • IPv6 network (requires Session Pooler)

Supabase (PostgreSQL)

  • Free tier: 500MB storage
  • Session Pooler on port 5432
  • Region: AWS ap-southeast-2
  • SSL required: sslmode=require
  • Connection via aws-1-... hostname

Key Problem Solved

⚠ Railway is IPv6-only. Supabase's direct connection (db.xxx.supabase.co:5432) only accepts IPv4. The fix: use the Session Pooler URL (aws-1-ap-southeast-2.pooler.supabase.com:5432) which is dual-stack and accepts both.
⚠ JPQL lower(bytea) error. PostgreSQL couldn't infer parameter type in LOWER(:search). The fix: LOWER(CONCAT('%', CAST(:search AS string), '%')) forces the type cast explicitly.
ℹ Sidebar sliding drawer implementation. The sidebar uses Tailwind's translate-x utilities with a 300ms ease-in-out transition. Mobile: fixed inset-y-0 left-0 z-30 w-72 -translate-x-full — opens to translate-x-0. A fading overlay (transition-opacity duration-300) and an × close button inside the drawer header let users dismiss it. Desktop (lg:): sidebar is relative and always visible — no drawer behaviour at all.
