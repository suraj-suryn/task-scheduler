1 Registration & Login

Creating an Account

  1. Go to /register or click "Create one" on the login page
  2. Fill in all four fields: Username, Email, Password, Confirm Password
  3. Click Register
  4. On success → redirected to login with a confirmation message
  5. Login with your new credentials

3 Categories

Categories let you organize tasks by color-coded groups (e.g. "Work", "Personal", "Urgent").

  1. Click Categories in the left sidebar
  2. Enter a category name in the input field
  3. Click the color swatch to pick a color (hex color picker)
  4. Click Add Category
  5. Your category appears as a colored card below

Deleting a Category

  • Click the ✕ button on any category card
  • The category is removed (existing tasks keep their category reference)
Tip: Create categories before creating tasks — you can then assign tasks to them from the task form.

4 Creating a Task

  1. Click My Tasks in the sidebar
  2. Click the + Add Task button (top right)
  3. A modal dialog opens — fill in the fields
  4. Click Save Task
  5. A green toast confirms the task was saved

Task Form Fields

FieldRequiredDescription
Task NameYesShort descriptive title
DescriptionNoLonger details about the task
PriorityYesNumber 1–10 (10 = highest). Stored in a Max-Heap internally.
StatusYesPENDING / RUNNING / COMPLETED / FAILED
Start TimeYesWhen the task begins (date & time picker)
End TimeYesWhen the task ends — must be after start time
CategoryNoPick from your created categories
RecurrenceNoNONE / DAILY / WEEKLY / MONTHLY
Recurrence End DateOnly if recurringStop auto-creating after this date
DependenciesNoComma-separated IDs of tasks that must complete first

Automatic Checks on Save

  • Overlap Detection — If another task already occupies the same time slot, save is blocked with an error toast. Uses an Interval Tree (O(log n)).
  • Cycle Detection — If adding the dependency would create a circular chain (A→B→C→A), save is blocked. Uses DFS graph traversal.

6 Recurring Tasks

Recurring tasks automatically create the next occurrence when the current one is completed.

  1. In the task form, set Recurrence to Daily, Weekly, or Monthly
  2. Set a Recurrence End Date (the last date a new occurrence will be created)
  3. Save the task normally
  4. When the scheduler marks it complete, a new task is created automatically with the next date shifted by the recurrence interval
Recurrence TypeNext occurrence created
DAILYStart + 1 day, End + 1 day
WEEKLYStart + 7 days, End + 7 days
MONTHLYStart + 1 month, End + 1 month
ℹ️ Note: The scheduler runs every 60 seconds. Tasks are auto-completed only when their end time is in the past.

7 Task Dependencies

You can specify that a task depends on other tasks being completed first.

  1. Find the task IDs you want as dependencies (shown in the task list table)
  2. In the task form's Dependencies field, enter comma-separated IDs: e.g. 1,3,5
  3. The system checks for circular dependencies automatically

Rules

  • Dependencies are stored as a directed graph (adjacency list)
  • If A depends on B, then B must complete before A
  • Circular chains like A→B→C→A are rejected at save time with an error

9 REST API

The API is available for developers to integrate or automate task management. All endpoints require authentication (same session cookie as the web UI). All responses follow the format:

{ "success": true, "data": { ... } }

Endpoints

MethodEndpointDescription
GET/api/tasksList your tasks (supports ?search=&status=&categoryId=)
POST/api/tasksCreate a new task
PUT/api/tasks/{id}Update a task fully
DELETE/api/tasks/{id}Delete a task
PATCH/api/tasks/{id}/statusUpdate status only — body: {"status":"COMPLETED"}
GET/api/tasks/dependenciesGet dependency graph as adjacency map
GET/api/dashboard/statsDashboard statistics as JSON

Example — Create Task

POST /api/tasks
Content-Type: application/json

{
  "name": "Write report",
  "priority": 8,
  "status": "PENDING",
  "startTime": "2026-06-10T09:00",
  "endTime": "2026-06-10T11:00"
}
